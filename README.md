# Gerenciamento de Permissões de Arquivos no Linux – Revisão de Autorização de Segurança

Este projeto apresenta uma revisão de permissões de sistema de arquivos realizada em um ambiente empresarial simulado, como parte de um programa de treinamento em segurança cibernética.

## Escopo

- Exame das permissões de arquivos e diretórios existentes
- Identificação de permissões mal configuradas ou excessivas
- Modificação de permissões para garantir a autorização adequada
- Validação das alterações para assegurar o controle de acesso seguro

## Cenário

Neste cenário, atuei como um profissional de segurança apoiando uma equipe de pesquisa dentro de uma grande organização. Minha responsabilidade foi verificar se os usuários tinham as permissões de acesso corretas e remover qualquer acesso não autorizado, a fim de manter a segurança do sistema.

## Atividades Principais

- Uso do comando ls -l para examinar as permissões atuais de arquivos e diretórios
- Interpretação das permissões de leitura (r), gravação (w) e execução (x) para proprietário, grupo e outros
- Identificação de inconsistências nas permissões
- Modificação de permissões usando chmod
- Verificação das alterações de permissão para garantir a implementação correta

## Comandos Utilizados

- ls -l
- chmod

## Conceitos de Segurança Aplicados

- Princípio do Menor Privilégio
- Controle de Acesso
- Gerenciamento de Autorizações
- Reforço da Segurança do Sistema de Arquivos

## Resultado

- Permissões não autorizadas foram removidas
- Níveis de acesso apropriados foram implementados
- Permissões do sistema de arquivos alinhadas aos requisitos de segurança
- Postura geral de segurança do sistema aprimorada
